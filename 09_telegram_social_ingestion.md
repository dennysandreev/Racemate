# 09. Telegram Social Ingestion через MTProto

## Назначение

RaceMate читает публикации из публичных и доступных приватных Telegram-каналов через отдельный пользовательский аккаунт. Парсер использует GramJS и общие таблицы `social_sources`, `social_posts`, `social_post_media`, `social_post_tags`, а публикация на сайте всегда проходит через `social.process_ai`.

Bot API webhook остаётся доступен для каналов, куда можно добавить бота. Для остальных каналов используется адаптер `telegram-mtproto`.

## Создание Telegram API application

1. Создайте отдельный Telegram-аккаунт RaceMate и включите для него 2FA.
2. Войдите на [my.telegram.org](https://my.telegram.org) этим аккаунтом.
3. Откройте `API development tools` и создайте application.
4. Скопируйте `api_id` и `api_hash` в локальные секреты:

```dotenv
TELEGRAM_API_ID=123456
TELEGRAM_API_HASH=replace-with-secret
```

Официальная инструкция: [Creating your Telegram Application](https://core.telegram.org/api/obtaining_api_id).

## Создание session string

Запустите команду в интерактивном терминале:

```bash
pnpm telegram:authorize
```

Команда скрыто запросит номер телефона, код входа и, если включена 2FA, пароль. После успешного входа она один раз покажет `TELEGRAM_SESSION=...`.

Добавьте значение в секреты окружения web/worker/cron:

```dotenv
TELEGRAM_SESSION=replace-with-session-string
```

Правила безопасности:

- не добавляйте `.env` в git;
- не вставляйте session string в админку или Supabase;
- не передавайте session string в аргументах команд;
- очищайте сохранённую историю терминала после авторизации, если она записывает вывод;
- обычный worker и cron никогда не запрашивают код или пароль интерактивно.

Официальное описание входа: [User Authorization](https://core.telegram.org/api/auth).

## Настройка окружения

Минимальная конфигурация:

```dotenv
TELEGRAM_API_ID=
TELEGRAM_API_HASH=
TELEGRAM_SESSION=
```

Операционные настройки:

```dotenv
TELEGRAM_FETCH_CONCURRENCY=2
TELEGRAM_ALBUM_SETTLE_SECONDS=30
TELEGRAM_EDIT_LOOKBACK_MESSAGES=100
TELEGRAM_MAX_MESSAGES_PER_SOURCE=2000
TELEGRAM_MEDIA_MAX_BYTES=52428800
```

- `TELEGRAM_FETCH_CONCURRENCY` ограничивает число одновременно обрабатываемых каналов.
- `TELEGRAM_ALBUM_SETTLE_SECONDS` не даёт отправить альбом в AI до завершения загрузки его частей.
- `TELEGRAM_EDIT_LOOKBACK_MESSAGES` задаёт глубину повторной проверки недавних публикаций на правки.
- `TELEGRAM_MAX_MESSAGES_PER_SOURCE` ограничивает один запуск для очень активных каналов.
- `TELEGRAM_MEDIA_MAX_BYTES` должен соответствовать меньшему из глобального лимита Supabase и лимита bucket `social-media`. Для проекта на Free используется 50 МБ; более крупное вложение пропускается, а текст публикации сохраняется.

## Подключение канала

В `/admin` добавьте отдельный источник для каждого канала:

- площадка: `Telegram`;
- подключение: `Telegram MTProto`;
- аккаунт или канал: публичный `@username` либо числовой ID;
- ссылка: публичная ссылка `https://t.me/channel` или доступная администратору ссылка на приватный канал;
- публикация: после AI-анализа или через модерацию; исходный текст не переписывается, AI определяет характер публикации и добавляет теги;
- доверие: официальный источник, СМИ или сообщество;
- первая загрузка: настраивается для каждого канала; для текущих источников RaceMate используется 14 дней;
- пересланные посты: включаются отдельно.

### Какие гоночные серии публикуются

AI определяет серию по содержанию каждой публикации, а не по названию или общей тематике канала.

Публикуются материалы, где основная тема — классические одноместные формульные серии с открытыми колёсами: Formula 1, Formula 2, Formula 3, Formula 4, F1 Academy, Formula Regional и другие национальные или международные формульные чемпионаты.

Не публикуются Formula E, Extreme E, WEC, Le Mans, IMSA, гонки на выносливость, GT, кузовные серии, ралли, MotoGP, NASCAR, IndyCar и другие категории вне целевой группы. Упоминание пилота или команды F1 в материале о другой серии само по себе не делает публикацию подходящей. Если серию нельзя уверенно определить, публикация отклоняется. В смешанном материале целевая формульная серия должна быть основной темой.

Приватный канал читается только тогда, когда аккаунт RaceMate уже является его участником. Для приватного канала без username используйте channel ID, например `-1001234567890`.

Кнопка `Проверить и загрузить` ставит в очередь синхронизацию выбранного источника. Для прямой проверки из терминала:

```bash
pnpm worker:social:telegram -- --source=<source-uuid>
```

## Запуск worker

Загрузить только Telegram:

```bash
pnpm worker:social:telegram
```

Загрузить все социальные источники и затем обработать новые записи:

```bash
pnpm worker:social
pnpm worker:social:ai
```

`docker-compose.yml` уже запускает `social.fetch_all` и `social.process_ai` в cron-контейнере. Источники обрабатываются независимо: ошибка одного канала сохраняется в `social_sources.last_error` и не останавливает остальные.

Telegram возвращает историю от новых сообщений к старым. При первой синхронизации worker идёт назад до настроенной даты, при следующих использует `last_seen_external_id`. Официальное описание метода: [messages.getHistory](https://core.telegram.org/method/messages.getHistory).

## Альбомы, медиа и правки

- одиночная публикация получает `external_id` вида `telegram:<channelId>:<messageId>`;
- альбом получает `telegram:<channelId>:album:<groupedId>`;
- части альбома сохраняются одной публикацией в исходном порядке;
- изображения и видео загружаются в bucket `social-media`;
- подпись альбома используется как исходный текст;
- изменение подписи обновляет существующую запись и возвращает её в AI-очередь;
- повторный запуск обновляет существующие записи и не создаёт дубли.

`raw_payload` содержит только безопасную техническую сводку. Session string, телефон, код и пароль никогда не записываются в БД.

## FLOOD_WAIT

Worker не скрывает долгий FLOOD_WAIT внутренним ожиданием. Он сохраняет безопасное сообщение и `rate_limited_until` для конкретного источника, после чего продолжает другие каналы. Следующий обычный запуск пропустит источник до окончания ограничения.

Не уменьшайте интервалы синхронизации без необходимости. Telegram отдельно предупреждает о лимитах и RPC-ошибках в [Error handling](https://core.telegram.org/api/errors).

## Обновление и отзыв авторизации

Если сессия истекла или была отозвана:

1. удалите старый `TELEGRAM_SESSION` из secret storage;
2. снова запустите `pnpm telegram:authorize`;
3. замените secret во всех worker/cron окружениях;
4. перезапустите сервисы;
5. запустите `pnpm worker:social:telegram` и проверьте `last_success_at`.

Если session string скомпрометирован, сначала удалите его из всех окружений. Затем завершите соответствующий сеанс в Telegram в разделе `Настройки -> Устройства` или отзовите остальные авторизации средствами Telegram API. Официальный метод полного отзыва: [auth.resetAuthorizations](https://core.telegram.org/method/auth.resetAuthorizations). После отзыва создайте новый session string.
