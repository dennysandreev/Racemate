# 07. Telegram Bot Spec

## Роль

Telegram-бот RaceMate пишет только тогда, когда пользователю действительно нужно внимание:
- напоминает о выбранных сессиях;
- сообщает, что результаты готовы;
- напоминает о незаполненном прогнозе;
- присылает выбранные новости с изображением и ссылкой на материал.

Уведомления о погоде, чемпионате, лигах, опросах и дневных дайджестах не входят в текущую Telegram-интеграцию.

## Настройки

### Новости

- `Все новости` — каждый новый материал после успешной AI-обработки;
- `Любимые пилоты` — новости с тегами избранных пилотов;
- `Любимая команда` — новости с тегами избранной команды.

Новость отправляется с изображением, русским заголовком, короткой сводкой, источником и кнопкой `Читать подробнее`. Если Telegram не может загрузить изображение, worker отправляет то же сообщение без него.

### Фентази

Единственная настройка — `Дедлайн по прогнозам`.

Для квалификационной и гоночной части отдельно:
- за 4 часа до дедлайна;
- за 15 минут до дедлайна;
- только если соответствующая часть прогноза ещё не заполнена.

Поул относится только к квалификационной части. Топ-10 и специальные прогнозы относятся к гоночной части.

### Сессии

Настройки задаются отдельно для:
- практик;
- спринт-квалификации;
- квалификации;
- спринта;
- гонки.

Для каждой группы доступны:
- включение или отключение уведомлений;
- напоминания за 24 часа, 1 час и 15 минут;
- режим `Без спойлеров`.

Если режим без спойлеров включён, сообщение после финиша сообщает только о готовности результатов и ведёт на сайт. Если режим выключен, сообщение показывает топ-3.

Изменение времени или отмена сессии отправляется только пользователям, у которых включена соответствующая группа сессий.

## Команды

- `/start` — подключение по одноразовой ссылке или приветствие;
- `/help` — доступные команды;
- `/next` — ближайшая сессия во времени пользователя;
- `/summary` — последняя дневная сводка по запросу пользователя;
- `/settings` — ссылка на настройки в RaceMate;
- `/unsubscribe` — отключение уведомлений.

Команда `/summary` остаётся ручной и не создаёт автоматический дайджест.

## Отправка

```text
Telegram подключён
→ категория включена
→ событие подходит под индивидуальные настройки
→ уведомление ещё не отправлялось
→ durable queue
→ Telegram API
→ журнал результата
```

- Все отправки идут через `notification_queue`.
- `dedupe_key` не позволяет повторить одно событие одному пользователю.
- Каждая запись отправляется независимо: ошибка одного сообщения не блокирует следующие.
- Временная ошибка повторяется с задержкой; после трёх попыток запись становится `failed`.
- Если пользователь заблокировал бота, `telegram_accounts.is_active` переключается в `false`.
- Сообщения отправляются сразу, без тихих часов и автоматических дайджестов.
- Текст использует Telegram HTML, эмодзи и inline-кнопки; данные из источников экранируются.
- Raw link token не хранится и не логируется.
- Telegram webhook принимает запросы только с настроенным secret token.

## Команды worker

```text
node worker/index.mjs notifications.enqueue
node worker/index.mjs notifications.dispatch
```

`notifications.enqueue` запускается после синхронизации календаря, результатов, новостей и фентази. `notifications.dispatch` можно запускать каждые 1–5 минут.
