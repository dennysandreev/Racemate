# 07. Telegram Bot Spec

## Роль

Telegram-бот RaceMate сопровождает гоночный уикенд и возвращает пользователя в продукт в важный момент:
- напоминает о старте сессии;
- помогает не пропустить дедлайн прогноза;
- сообщает о результатах;
- предупреждает о заметной погоде и переносах;
- присылает важные новости и дневную сводку;
- сообщает об обновлении чемпионата.

Лиги, соревнования с друзьями и опросы не входят в Telegram-интеграцию.

## Обязательное правило без спойлеров

- Практики: топ-3 можно показать сразу.
- Квалификация, спринт и гонка: первое сообщение не содержит имён, позиций, изображения или превью, раскрывающего результат.
- Результат открывается только после нажатия «Показать результаты».
- До раскрытия скрываются связанные фэнтези-очки, изменения чемпионата и заголовки новостей, которые могут раскрыть итог.
- Раскрытие хранится отдельно для пользователя и сессии.

## Команды

- `/start` — подключение по одноразовой ссылке или приветствие;
- `/help` — доступные команды;
- `/next` — ближайшая сессия во времени пользователя;
- `/summary` — последняя дневная сводка;
- `/settings` — ссылка на настройки в RaceMate;
- `/unsubscribe` — отключение уведомлений.

## Отправка

Каждое событие проходит проверки:

```text
подключён ли Telegram
→ включена ли категория
→ не заполнен ли уже прогноз
→ не действуют ли тихие часы
→ нужно ли скрыть спойлер
→ не отправлялось ли такое сообщение
→ очередь
→ Telegram API
→ журнал результата
```

- Все отправки идут через `notification_queue`.
- `dedupe_key` не позволяет повторить одно событие одному пользователю.
- Ошибки временно повторяются с задержкой.
- После трёх ошибок задача становится `failed`.
- Если пользователь заблокировал бота, `telegram_accounts.is_active` переключается в `false`.
- Raw link token не хранится и не логируется.
- Telegram webhook принимает запросы только с настроенным secret token.

## Команды worker

```text
node worker/index.mjs notifications.enqueue
node worker/index.mjs notifications.dispatch
```

Первую команду нужно запускать после синхронизации календаря, результатов, погоды, новостей и фэнтези. Вторую можно запускать каждые 1–5 минут.
